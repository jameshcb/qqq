# 下窗开场词模板

> **开场词 = 指针不是容器,状态只活在 HANDOFF,开场词只指过去。**
> 填好后写进 `/tmp/qqq-handoff-<项目名>-<HHMM>.txt`(时分防同项目双窗互覆),由 SKILL Step 6 用 vscode 深链接预填进新标签页(pbcopy 不可靠,已弃用)。内容必须自包含、可直接粘贴 —— **handoff 较长时 Step 6 会降级成预填「读这份文件」短指针,新窗直接读它,所以自包含是硬约束,不能写残篇**。
>
> **软上限 ≤1KB / ≤25 行**(类比 HANDOFF ≤100 行)。超了 = 你在往开场词里抄 HANDOFF/CLAUDE 已有的东西,删掉它们、只留指针。
>
> 下面以 **TOPS 为例**;本模板**全局通用**,换项目时按当前项目替换:开头项目名、读哪些文件、§N 都因项目而异。

```
🪟 2026-06-16 14:30 开窗 · 接 <项目> · 本窗:<中文一句:新窗要干的事(下一步)>
先读(别急动手):
1. git fetch + log HEAD..origin/main + status + branch;多窗复验 git(详 sessions 今天复盘)
2. HANDOFF §1-3   3. docs/sessions/<今天>.md 扫 ⚠️🎓   4. decisions 相关 §N
当前状态:<1-2 句,只说最新落点 + 指 HANDOFF §1>
下一步(跟我确认):▶… ▶… ▶…(详 HANDOFF §2/§3)
纪律:守 CLAUDE §N + 红线;本窗新教训 = <一句>;待决 = <一句>
```

## 填写要点(指针不是容器)

- **第一行 = 带日期时间的中文主题锚**:`🪟 <YYYY-MM-DD HH:MM> 开窗 · 接 <项目> · 本窗:<中文主题一句>`。时间 = 开窗当刻(`date '+%Y-%m-%d %H:%M'`);主题句概括**新窗要干的事(下一步)**,是主题短语**不是状态复述**(禁 commit hash);`🪟` 作视觉锚 —— 新窗第一屏被英文 claude-mem 上下文淹没,这行让 James 一眼认出何时开窗、干什么
- **「当前状态」硬性 1-2 句** —— 只说最新落点 + 指 HANDOFF §1;**禁内联 commit hash 流水账**、禁把 HANDOFF §1/§2 内容抄进来。状态权威在 HANDOFF,开场词漂了反而误导下窗
- **禁重贴 CLAUDE/HANDOFF 已有内容** —— 纪律一行带过(「守 CLAUDE §N + 红线」),下窗本来就读 CLAUDE.md,别整段重贴 §12 / 红线 / 工具优先级
- **开场词只允许在指针之外额外承载两样**:① 本窗新教训(一句)② 待决(一句)。这两样是「过去」的产物、HANDOFF 里未必有,值得直接带;其余一律指过去
- **§N 要具体**:写 §41 不写"看 decisions";下个 Claude 才不用全文扫
- **下一步是"选项"不是"指令"**:让下个 Claude 跟 James 确认,不替他拍板
